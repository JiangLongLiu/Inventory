## 环境搭建问题

### React Native开发环境配置失败怎么办？

当配置React Native开发环境时，如果遇到依赖安装失败或构建错误，请首先检查`package.json`文件中指定的Node.js版本。本项目要求使用Node.js 18.15.0版本，版本不匹配可能导致依赖安装失败。

确保安装了正确的Node.js版本后，运行`yarn install`安装所有依赖。对于iOS开发，需要在`ios/`目录下运行`bundle install`安装Fastlane，并执行`yarn run pod-install`安装CocoaPods依赖。

对于Android开发，确保已正确配置Gradle签名变量。在`~/.gradle/gradle.properties`中添加以下内容：
```
INVENTORY_APP_UPLOAD_STORE_FILE=/path/to/your/keystore
INVENTORY_APP_UPLOAD_KEY_ALIAS=your-key-alias
INVENTORY_APP_UPLOAD_STORE_PASSWORD=your-store-password
INVENTORY_APP_UPLOAD_KEY_PASSWORD=your-key-password
```

## 数据同步问题

### PouchDB与CouchDB同步失败如何排查？

PouchDB与CouchDB同步失败通常由认证、网络或配置问题引起。项目中的`DBSyncManager.tsx`文件实现了详细的同步管理逻辑，包括错误日志记录和会话管理。

同步失败的常见原因及解决方案：
1. **认证失败**：确保用户名和密码正确，并且CouchDB服务器已正确配置用户权限。代码中使用`logIn`方法重置会话，避免因相同主机名使用不同凭据导致的问题。
2. **网络连接问题**：检查服务器URI是否可达，防火墙是否阻止了相应端口。
3. **权限不足**：确认用户具有访问数据库的权限，错误信息"missing"通常表示权限不足。
4. **序列号冲突**：同步过程中使用序列号（seq）跟踪变更，确保本地和远程数据库的序列号同步。

同步过程中，系统会记录详细的HTTP错误日志，包括状态码和响应体，便于诊断问题。`retry: true`选项确保在网络不稳定时自动重试。

## RFID模块连接与读取问题

### RFID设备无法连接怎么办？

RFID设备连接问题可能由硬件、蓝牙或配置问题引起。项目支持两种RFID设备类型：内置读取器和BLE设备，可通过`RFIDUHFModuleScreen.tsx`中的切换开关选择。

常见连接问题及解决方案：
1. **设备未开启**：确保RFID设备已开启并处于可发现状态。
2. **蓝牙权限**：检查应用是否具有蓝牙权限，特别是在Android设备上。
3. **设备类型选择错误**：在应用中正确选择设备类型（内置读取器或BLE）。
4. **初始化失败**：查看日志中的初始化错误信息，可能是驱动或硬件问题。

代码中通过`receiveConnectStatusChange`回调函数监控连接状态变化，并提供相应的用户界面反馈。

### RFID标签读取失败如何处理？

标签读取失败可能由以下原因造成：
1. **功率设置过低**：在读取界面调整发射功率（Power），适当增加功率可能改善读取效果。
2. **内存银行选择错误**：确保选择了正确的内存银行（EPC、TID或USER）。
3. **访问密码错误**：如果标签受保护，需要提供正确的访问密码。
4. **过滤器配置不当**：检查过滤器设置是否正确，包括内存银行、位偏移和数据匹配。

读取操作的代码实现了完整的错误处理机制，捕获并处理各种异常情况，确保应用不会因读取失败而崩溃。

## CSV导入导出格式与错误

### CSV导入导出的格式要求是什么？

CSV导入导出功能通过`csvRowToItem.ts`和`itemToCsvRow.ts`文件实现，支持以下字段：

| 字段名 | 说明 | 格式要求 |
|-------|------|---------|
| ID | 项目唯一标识 | UUID格式 |
| Collection Ref. No. | 集合参考编号 | 以单引号开头的字符串 |
| Reference No. | 项目参考编号 | 以单引号开头的字符串 |
| Serial | 序列号 | 文本 |
| Name | 名称 | 文本 |
| Notes | 备注 | 文本 |
| Model Name | 型号名称 | 文本 |
| Purchase Price | 购买价格 | 数字，自动转换为小数 |

导出的CSV文件会包含所有项目信息，而导入时只需提供必要的字段。系统会自动处理价格字段的小数转换。

### CSV导入常见错误有哪些？

CSV导入常见错误及解决方案：
1. **文件格式错误**：确保上传的是有效的CSV文件，第一行应包含正确的列标题。
2. **必填字段缺失**：Name字段是必填的，缺少该字段的行将被忽略。
3. **参考编号不匹配**：如果指定了集合参考编号，但系统中不存在对应的集合，系统会尝试自动查找。
4. **数据验证失败**：导入的数据需要通过验证，如价格必须为有效数字。

导入过程中，系统会收集所有错误信息并在完成后统一显示，便于用户修正后重新导入。匹配ID的项目将被更新，而非重复创建。

## 性能优化建议

### 大量数据加载时如何优化性能？

当处理大量数据时，可采用以下性能优化策略：

1. **分批处理**：使用`useDataCount`和分页功能，避免一次性加载所有数据。通过设置`skip`和`limit`参数实现分页加载。
2. **延迟加载**：对于非关键数据，使用`lowPriority`标志将其放入低优先级队列（LPJQ）中处理，避免阻塞主线程。
3. **数据缓存**：利用PouchDB的本地存储优势，减少网络请求。同步时使用`retry: true`和适当的批处理大小（`batch_size`）。
4. **异步操作**：将耗时操作（如数据一致性检查）放在异步循环中执行，并提供进度反馈和取消功能。
5. **内存优化**：避免在内存中保留大量数据引用，及时释放不再需要的对象。

数据一致性检查功能（`FixDataConsistencyScreen.tsx`）展示了如何处理大量数据：使用生成器函数逐批处理数据，每处理一批后主动让出控制权，确保UI能够响应。
