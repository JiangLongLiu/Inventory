## 简介
本指南旨在为新开发者提供在本地环境中设置和运行Inventory项目的完整说明。Inventory是一个基于RFID技术的资产管理系统，适用于家庭和企业场景。本指南将指导您完成从环境配置到应用运行的全过程，确保您能够顺利开始开发工作。

## 环境准备
在开始开发之前，您需要准备以下开发环境：

### Node.js
项目要求Node.js版本为18.15.0。建议使用版本管理工具（如nvm）来管理Node.js版本，以确保与项目要求一致。

### Yarn
项目使用Yarn作为包管理器。请确保已安装Yarn，并建议使用与项目兼容的版本。

### React Native CLI
安装React Native CLI工具，用于构建和运行React Native应用：
```bash
npm install -g @react-native-community/cli
```

### iOS开发环境（Xcode）
对于iOS开发，您需要：
- 安装Xcode（建议最新稳定版本）
- 安装Xcode命令行工具
- 配置Apple开发者账户（用于在物理设备上运行应用）

### Android开发环境（Android Studio）
对于Android开发，您需要：
- 安装Android Studio
- 配置Android SDK
- 设置ANDROID_HOME环境变量
- 安装必要的SDK组件（包括Android SDK Platform-Tools和Android SDK Build-Tools）

## 依赖安装
项目采用多包结构，需要在多个目录下安装依赖。

### 使用自动化脚本安装所有依赖
项目提供了一个自动化脚本`scripts/yarn-install-all.sh`，用于安装所有子项目的依赖：
```bash
bash scripts/yarn-install-all.sh
```

该脚本会依次进入以下目录并执行`yarn install`：
- `Data/`
- `packages/epc-utils`
- `packages/data-storage-couchdb`
- `packages/integration-airtable`
- `packages/integration-snipe-it`
- `App/`

### 脚本作用详解
`scripts/yarn-install-all.sh`脚本的主要作用是简化多包项目的依赖管理。它通过遍历项目中的各个子包目录，自动执行依赖安装，避免了手动逐个目录安装的繁琐过程。这是项目根目录`package.json`中`postinstall`钩子调用的脚本，确保在主项目安装后自动完成所有子项目的依赖安装。

## iOS平台构建与运行
### 开发环境配置
1. 复制配置文件：
```bash
cp App/ios/config.xcconfig.sample App/ios/config.xcconfig
```
2. 编辑`App/ios/config.xcconfig`文件，根据您的开发环境修改配置，特别是`DEVELOPMENT_TEAM`（您的Apple开发者团队ID）和`PRODUCT_BUNDLE_IDENTIFIER_*`（应用包标识符）。

### 安装iOS原生依赖
使用CocoaPods安装iOS原生依赖：
```bash
cd App
yarn run pod-install
```
此命令会执行`App/scripts/pod-install.sh`脚本，该脚本会：
- 更新hermes-engine pod
- 执行`pod install`安装所有依赖

### 在Xcode中运行
1. 打开Xcode工作区：
```bash
open App/ios/Inventory.xcworkspace
```
2. 在Xcode中选择合适的scheme（如Inventory、Inventory (Dev)或Inventory (Nightly)）和目标设备
3. 点击"Run"按钮或使用快捷键Cmd+R构建并运行应用

## Android平台构建与运行
### Gradle配置
确保Android项目的Gradle配置正确。`App/android/gradle.properties`文件中包含重要配置：
- `hermesEnabled=true`：启用Hermes JavaScript引擎
- `reactNativeArchitectures`：指定支持的CPU架构
- `AsyncStorage_db_size_in_MB=512`：设置AsyncStorage数据库大小

### 构建与运行
在项目根目录执行以下命令来运行Android应用：
```bash
cd App
yarn run android
```
此命令会：
1. 启动Metro开发服务器
2. 构建Android应用
3. 在连接的设备或模拟器上安装并启动应用

如果这是首次运行，Gradle将下载所有必要的依赖，这可能需要一些时间。

### 签名配置
对于发布版本，需要配置应用签名。在`~/.gradle/gradle.properties`中添加：
```
INVENTORY_APP_UPLOAD_STORE_FILE=/path/to/your/keystore
INVENTORY_APP_UPLOAD_KEY_ALIAS=your-key-alias
INVENTORY_APP_UPLOAD_STORE_PASSWORD=your-store-password
INVENTORY_APP_UPLOAD_KEY_PASSWORD=your-key-password
```

## 常见问题排查
### 原生依赖安装失败
#### iOS问题
- **CocoaPods安装失败**：确保已正确安装CocoaPods，并尝试执行`pod repo update`更新本地仓库
- **Hermes引擎问题**：脚本中包含`pod update hermes-engine --no-repo-update`命令，用于解决本地Podspec版本不匹配问题

#### Android问题
- **Gradle同步失败**：检查网络连接，确保可以访问Maven中央仓库和Google Maven仓库
- **Hermes编译错误**：确认`hermesEnabled`设置正确，并尝试清理项目：`cd android && ./gradlew clean`

### 模拟器连接问题
- **设备未检测到**：运行`adb devices`检查设备连接状态
- **Metro服务器连接失败**：确保设备与开发机在同一网络，并尝试在应用内摇晃设备打开开发者菜单，选择"Change bundle location"设置正确的IP地址和端口

### 构建错误
- **Bitcode相关错误**：项目Podfile中包含bitcode strip脚本，用于移除框架中的bitcode
- **签名错误**：确保在Xcode中正确配置了开发团队和代码签名设置
